For each software used in the final project, the following information is recorded.
This file is updated each time a new tool is introduced.

---

| Software | Description | Strengths | Weaknesses | Assumptions | User Choices |
|---|---|---|---|---|---|
| **FastQC** | Quality control tool for raw Illumina sequencing reads. Generates an HTML report covering per-base quality scores, GC content, adapter contamination, and duplication levels | Fast; no configuration needed; visual HTML output; works on any FASTQ file | Does not trim or fix reads — only reports problems; some modules give false warnings for high-GC organisms like *M. tb* | Assumes standard Illumina Phred+33 encoding; paired-end files run separately | `--threads` (CPU cores); `--outdir` (output directory) |
| **SRA Toolkit (`fasterq-dump`)** | NCBI tool for downloading raw sequencing data from the Sequence Read Archive. Faster, multithreaded successor to `fastq-dump` | Direct access to all public NCBI/ENA data; parallel downloads; correctly separates paired-end reads | Large downloads require significant disk space; requires `vdb-config` setup; conda install fails on Apple Silicon (osx-arm64) — must install binary directly from NCBI | Requires internet and valid SRA accession; sufficient local disk space | `--split-files` (separate R1/R2 for paired-end); `--threads` (parallelization) |
| **gzip** | Standard Unix compression tool. Compresses .fastq files to .fastq.gz format | Universally available; reduces file size 4–6x; .fastq.gz accepted by all major bioinformatics tools | Slow for large files; not parallelized natively | Lossless — no data lost | Wildcard expansion (`*.fastq`) to compress all files at once |
| **BWA MEM** | Burrows-Wheeler Aligner — maps short Illumina reads to a reference genome using the Burrows-Wheeler transform. Optimized for reads 70bp–1Mbp | Fast and accurate for Illumina data; handles paired-end reads; widely used and validated in bacterial genomics | Struggles with highly repetitive regions; requires indexed reference genome before use | Short Illumina reads; reference genome available; reads evolved from the reference species | `-t` (threads); `-R` (read group string); paired-end files provided as R1 and R2 |
| **SAMtools** | Suite of tools for manipulating SAM/BAM alignment files including sorting, indexing, consensus generation, and basic statistics | Extremely fast; handles very large files; output compatible with all downstream tools | Requires knowledge of SAM/BAM format; limited variant calling alone | Valid SAM/BAM input format | `sort` (coordinate sort before indexing); `index` (creates .bai file); `consensus -f fasta` (generate per-sample consensus sequence) |
| **bcftools** | Toolkit for variant calling and VCF/BCF file manipulation. Used via `mpileup` + `call` pipeline for SNP calling | Fast; produces standard VCF output; supports filtering, merging, and indexing of variant files | More basic than GATK for complex variant calling; may miss low-frequency variants | Reference genome available; reads properly mapped and sorted | `mpileup -f` (specify reference); `call -mv` (variant sites only, multiallelic model) |
| **MUSCLE** | Multiple sequence alignment tool using iterative refinement algorithm. Attempted for whole-genome alignment of 8 *M. tb* consensus sequences | Fast iterative refinement; widely used for protein and gene-level alignments | **Fatal limitation encountered:** MUSCLE v5.3 cannot perform global alignment on sequences longer than ~1 million base pairs. Both `-align` and `-super5` modes failed with error "Too long, not appropriate for global alignment" on 4.4 million bp sequences. Not suitable for whole bacterial genome alignment | Sequences must be within length limits for global alignment mode | `-align` (standard mode); `-super5` (attempted for long sequences — also failed) |
| **MAFFT** | Multiple sequence alignment tool using iterative refinement algorithm (FFT-NS-i or L-INS-i strategies). Selected as replacement for MUSCLE after MUSCLE failed on whole-genome sequences | Handles very long sequences with `--auto` flag; automatically selects best strategy based on input; faster than MUSCLE for large inputs; same iterative refinement approach as MUSCLE | Slower than reference-based SNP approaches for whole genomes; alignment of 4.4 Mb sequences is memory intensive | Sequences are homologous (same species); nucleotide input specified with `--nuc` | `--auto` (automatic strategy selection based on data size); `--nuc` (nucleotide sequences); `--thread 4` (parallelization) |
| **ape** (R package) | Analysis of Phylogenetics and Evolution. Core R package for reading, manipulating, and plotting phylogenetic trees and DNA sequence data | Comprehensive tree manipulation; reads Newick/NEXUS/FASTA; integrates tightly with phangorn; RF distance calculation | Limited tree inference on its own — primarily NJ and distance methods | Sequences must be aligned before input; distance model must be appropriate for the data | `dist.dna(model="TN93")` — Tamura-Nei 1993 model chosen for accuracy with high-GC bacterial genomes; `pairwise.deletion=TRUE` for handling missing data; outgroup: ERR4796950 (Lineage 1) |
| **phangorn** (R package) | Phylogenetic analysis in R. Extends ape with maximum parsimony, maximum likelihood, and model testing capabilities | Multiple methods (MP, ML, NJ) in one package; tight ape integration; parsimony score output | Slower than standalone ML tools (e.g. IQ-TREE) for large datasets; whole-genome parsimony is computationally intensive | SNP sites evolve independently; all substitutions equally likely under unweighted parsimony; fewest changes preferred | `optim.parsimony(nj_tree, dna_phyDat)` — NJ tree used as starting topology; SPR rearrangements for optimization |
| **Neighbor Joining (NJ)** | Distance-based tree algorithm (Saitou & Nei, 1987). Iteratively joins taxa with the smallest corrected pairwise evolutionary distance | Very fast; corrects for multiple substitutions; good exploratory tree; no model parameters to optimize | Produces only one tree; sensitive to substitution model choice; less statistically rigorous than ML or Bayesian | Sequences evolved under the chosen substitution model (TN93); approximately equal evolutionary rates across lineages | Substitution model: TN93; outgroup for rooting: ERR4796950 (Lineage 1 Indo-Oceanic — most ancestral lineage) |
| **Maximum Parsimony (MP)** | Character-based tree method (Fitch, 1971). Finds the topology requiring the fewest total evolutionary changes to explain observed sequence data | Intuitive and model-free; no substitution model required; conceptually simple; good for comparison with distance methods | Susceptible to long branch attraction (LBA); slow for large datasets; inconsistent under some evolutionary conditions (Felsenstein zone) | SNP sites evolve independently; all substitutions equally likely (unweighted); evolution proceeds by fewest changes | Starting tree: NJ topology; `rearrangements="SPR"` for thorough search; outgroup: ERR4796950 |
